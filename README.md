# Wattle Framework

WattleFlow is a lightweight framework intended for `ETL` developers (Data Engineers). The framework is based on design patterns and uses a small footprint providing flexibility to build robust and bespoke ETL pipelines for data extraction, transformation and wrangling, as well as loading (storing) data.

The framework is designed with a `vendor-agnostic` approach and it has very minimal dependencies. WattleFlow framwork will ensure you have control over your security posture while giving you enough flexibility to develop bespoke libraries that meet your project's specific `security requirements`. The framwork code can run on your own machine, on-premises, and in the cloud.

## How the framework is tested
The framework has been tested on commands that collected data from online websites. PDF, CSV, and other data formats were used for this purpose. The data was then extracted, transformed, and stored in CSV files and databases. Examples can be found in the `examples` folder.

## Examples
Enclosed examples will help you learn how to use framwork and how to develop your own `extraction`, `transformation` and `load` commands. Demo will show how to use framework for your own needs. Examples how to utilise YAML to create your ETL pipeline will help you write simple builiding blocks that ensure fast development of reusability of your code when crafting bespoke piplines.

## Documentation
The basic framework documentation for creating wattle `commands` and `etl` piplines can be found in `documentation` folder.
